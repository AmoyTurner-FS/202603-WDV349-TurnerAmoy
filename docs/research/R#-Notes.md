## Topic - Wireframes & Application Structure

This research focuses on understanding how wireframes help plan the structure and layout of an application before development begins. Wireframes allow developers and stakeholders to visualize how users will interact with the application and how key features will be organized across different pages.

<br>

## Sub-Topic 1 - Wireframing Basics

Wireframes are low-fidelity layouts used to map out the structure of an application.

* Key finding 1: Wireframes focus on layout and structure rather than colors or visual design.
* Key finding 2: They help teams understand how users will navigate between different pages or features.
* Key finding 3: Wireframes can be created using simple sketches or tools like Figma.

<br>

## Sub-Topic 2 - Application Page Structure

Planning page structure helps organize features and determine how users will interact with the system.

* Key finding 1: Key features from the proposal should be mapped to specific pages in the app.
* Key finding 2: Static pages such as home, about, and contact should also be included in the wireframe planning.
* Key finding 3: Each page should clearly support a specific user goal or action.

<br>

## Sub-Topic 3 - Low Fidelity Design

Low-fidelity wireframes are used early in the design process before high-fidelity mockups are created.

* Key finding 1: Low-fidelity designs help teams focus on functionality instead of visuals.
* Key finding 2: They make it easier to test ideas quickly and adjust layouts if needed.
* Key finding 3: Tools like Figma allow interactive wireframes that simulate navigation.

## Wireframing Research

Wireframing helps plan the layout of the application before development begins. It allows developers to visualize how users will interact with the system and where key features will be located. For the CarFinder project, wireframes were created to map out the main interface, including search filters and results display.

Key findings:
- Wireframes should focus on layout and functionality rather than detailed design.
- Low-fidelity wireframes help quickly test layout ideas.
- Creating wireframes early helps clarify how users will navigate the application.


## Branding Research

Branding defines the overall look and feel of the application. This includes choosing colors, fonts, icons, and images that fit the target audience. For the CarFinder project, the goal is to create a darker, premium-style theme that fits classic and JDM car culture.

Key findings:
- Consistent colors and typography help create a recognizable brand.
- Dark themes can create a modern and premium feel.
- Icons, images, and UI elements should follow the same visual style.
