## Topic - GitHub Feature Branch Workflow

This document contains notes about GitHub feature branch workflow and how development work is organized when using Git.

<br>

## Sub-Topic 1 - Feature Branch Workflow
Feature branch workflow is a way developers organize their work when using Git. Instead of making changes directly on the main branch, developers create a separate branch for each task or feature. This helps keep the main project stable while development is happening. 

* New work is created in separate feature branches
* Each branch focuses on a specific change or task
* The work is merged back into another branch when it is finished 

<br>

## Sub-Topic 2 - Pull Requests
Pull requests are used to merge changes from one branch into another. They allow developers to review code before it becomes part of the main project. Pull requests also help track what changes were made and when they were added. 

* Pull requests allow code to be reviewed before merging
* They help track development changes
* Pull requests are used to merge feature branches into development branches

<br>

## Sub-Topic 3 - Branch Structure
In this course we are using multiple branches to organize development. Each branch has a different role in the workflow. 

* Feature branches are used for development work
* The dev branch collects development changes
* The stage branch is used to test work before it is finalized
